# Specs Workspace (v3)

Dette er arbeidsmappen for inkrementelle specs for Investment Report App v3.

## Hvordan bruke

1. Kopier `000-template.md` til neste ledige nummer.
2. Fyll ut scope, akseptansekriterier og testplan.
3. Få spec godkjent før implementasjon.
4. Implementer kun `In scope`.
5. Kjør kvalitetssjekker (lint/typecheck/test/build).

## Nummerering

- `000-template.md` — mal
- `001-v3-shell-navigation.md` — første aktive spec
- `002-...` osv.

## Regler

- Én feature/epic per spec.
- Ha tydelig `Out of scope`.
- Akseptansekriterier må være testbare.
- Beskriv loading/empty/error states for UI-arbeid.
